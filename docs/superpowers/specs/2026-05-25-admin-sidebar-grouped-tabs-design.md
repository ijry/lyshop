# 管理后台左侧导航分组 TAB 改造设计（Dashboard 不归组）

## 1. 背景与目标

当前管理后台左侧导航为单列深色菜单，存在两个问题：

1. 视觉上深色侧栏与浅色滚动条不协调。
2. 菜单层级弱，功能模块增长后查找效率下降。

本次目标：

- 采用“双列侧栏 + 一级分组 TAB”提升层级与可发现性。
- 支持 `hover 预览 + click 锁定` 的分组切换交互。
- `Dashboard` 固定入口，不归属于任何分组。
- 优先升级现有接口 `GET /admin/api/menus`，不新增接口。

## 2. 范围

### 2.1 In Scope

- 管理后台左侧导航 UI/交互改造。
- `GET /admin/api/menus` 返回结构升级为分组结构。
- 前端兼容旧接口结构（降级渲染）。
- 深色滚动条主题统一。
- 文档同步 `docs-site`。

### 2.2 Out of Scope

- RBAC 权限模型重构。
- 新增菜单编辑器或后台菜单配置页面。
- 非管理后台（`web`、`app`）导航改造。

## 3. 信息架构与交互

### 3.1 一级分组

建议固定键（后端返回，前端按顺序渲染）：

- `product` 商品
- `order` 订单
- `user` 用户
- `system` 系统
- `marketing` 营销
- `wms` 仓储
- `other` 其他

### 3.2 双列侧栏结构

- 列 A（分组列）：窄栏，仅显示一级分组 TAB。
- 列 B（菜单列）：展示当前分组的二级/三级菜单。
- `Dashboard`：固定显示在菜单列顶部，不随分组切换。

### 3.3 交互规则

- Hover 一级 TAB：右侧菜单列临时预览该分组。
- Click 一级 TAB：锁定该分组。
- 鼠标移出分组列：
  - 若已锁定，维持锁定分组。
  - 若未锁定，回退至当前路由所属分组。
- 当前路由高亮：分组 TAB 与菜单项同步高亮。

## 4. 接口升级设计（兼容升级）

### 4.1 现有接口

- 路径：`GET /admin/api/menus`
- 处理方式：升级返回结构，不新增新接口。

### 4.2 新返回结构（目标）

```json
{
  "dashboard": {
    "title": "Dashboard",
    "path": "/dashboard"
  },
  "groups": [
    {
      "key": "product",
      "title": "商品",
      "icon": "box",
      "sort": 10,
      "menus": [
        {
          "title": "商品管理",
          "icon": "box",
          "path": "/product",
          "sort": 10,
          "children": [
            { "title": "商品列表", "path": "/product/list" }
          ]
        }
      ]
    }
  ]
}
```

### 4.3 兼容策略

- 前端优先识别 `groups` 字段进入新渲染。
- 若响应仍为旧数组结构，则自动降级为旧单列渲染。
- 兼容期结束后可移除旧渲染分支（单独评估）。

## 5. 前端实现设计

### 5.1 文件与职责

- `admin/src/api/auth.ts`
  - 新增菜单响应类型定义。
  - `getMenus()` 返回类型从 `any[]` 升级为联合类型（新结构/旧结构）。

- `admin/src/layouts/AdminLayout.vue`
  - 侧栏模板重构为“双列结构 + Dashboard 固定入口”。
  - 新增状态：`hoverGroupKey`、`lockedGroupKey`、`activeGroupKey`。
  - 新增逻辑：根据当前路由反查所属分组。
  - 增加滚动条深色主题样式（仅作用于侧栏容器）。

- `admin/src/mock/index.ts`
  - Mock 的 `GET /admin/api/menus` 改为新结构，便于联调与验收。

### 5.2 状态优先级

`activeGroupKey` 计算优先级：

1. `hoverGroupKey`（正在预览）
2. `lockedGroupKey`（点击锁定）
3. `routeMatchedGroupKey`（当前路由匹配）
4. 默认分组（首个非空分组）

## 6. 错误处理与降级

- 接口异常：展示 `Dashboard` 固定入口 + 空菜单占位。
- 空分组：分组列可显示但菜单列显示“暂无菜单”。
- 路由无归属分组：自动切入 `other`，不存在则回落首个分组。

## 7. 测试与验收

### 7.1 功能验收

- Hover 分组可预览菜单。
- Click 分组可锁定菜单。
- 移出分组列后逻辑符合锁定/未锁定规则。
- Dashboard 始终可见且不归组。
- 刷新页面后分组与当前路由高亮正确。

### 7.2 视觉验收

- 侧栏与滚动条均为深色体系。
- 分组 TAB、菜单项、hover/active 状态层次清晰。

### 7.3 回归点

- 现有菜单路由跳转正常。
- 权限裁剪后的菜单显示正常。
- 旧接口结构返回时可正常降级。

## 8. 文档与部署影响

### 8.1 docs-site 同步要求

需更新 `docs-site` 对应后台导航文档，至少覆盖：

- 功能说明：双列分组导航与交互行为。
- 接口变化：`GET /admin/api/menus` 新响应结构与兼容说明。
- 部署影响：建议前后端同版本发布；若分步发布，依赖前端降级逻辑保障。

### 8.2 部署建议

- 推荐顺序：先发前端兼容版，再发后端新结构；或前后端同批次发布。

## 9. 实施边界与决策记录

- 决策 1：采用后端返回分组结构。
- 决策 2：`Dashboard` 固定入口，不归组。
- 决策 3：切换行为采用 `hover 预览 + click 锁定`。
- 决策 4：优先升级现有接口，不新增接口。
