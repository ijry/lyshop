# 演进记录

本页整理 AI 生图能力在管理后台中的演进轨迹与实践要点。

## 演进主线

- 初期：AI 能力以独立功能形态被感知。  
- 当前：AI 生图能力下沉到商品编辑工作流中，形成更短链路。  
- 结果：减少跨菜单跳转，素材生产与商品编辑形成闭环。  

## 为什么这样演进

- 商品素材生产与商品上架天然耦合，放在同一工作流可降低上下文切换成本。
- 运营同学在单页面内即可完成“生成-筛选-应用”，减少重复操作。
- 能力入口贴近业务动作，更容易建立标准化使用习惯。

## 实践要点

- 先确认素材用途（封面/轮播/详情）再生成，降低返工。
- 先选模型再写 Prompt，避免能力与目标错配。
- 关键素材保留任务记录，便于复盘、复用与审计。

## 文档边界

- 本页为文档层演进说明，不替代接口契约定义。
- 接口变化请以 API 文档章节为准。
